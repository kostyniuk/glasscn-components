Carving a Ring with mask-composite

The problem it solves

You have a pill with a perfect rounded corner from border-radius, and a gorgeous layered-gradient rim from lesson 0003 — but it's filling the entire pill, not just the edge. You need it to show only as a 1.5 px frame that hugs the rounded corners exactly.

The obvious tools all fail here:

  • border: 1.5px solid … — a border takes a colour, not a multi-stop gradient. Dead end.
  • border-image — does take a gradient, but it ignores border-radius; the gradient border renders square-cornered over your rounded pill. Useless for a glass pill.
  • A second, smaller element stacked on top to "cover the middle" — fragile, needs pixel-matched radius and a background to cover with, and breaks the moment the glass is translucent (you'd see the cover).

What you actually want: keep the gradient paint, but erase the interior, leaving the border band — and have the erase follow border-radius for free. That's a job for a mask.

The idea: a mask is a stencil, and two of them can be subtracted

A mask is an alpha stencil laid over an element: where the mask is opaque, the element's paint shows; where the mask is transparent, the paint is erased. (CSS luminance/alpha detail aside — think "opaque keeps, clear erases.")

The trick uses two mask layers, both just a solid opaque fill — linear-gradient(#000 0 0) is the idiom for "constant black, fully opaque, everywhere." On its own it's a verbose way to write "opaque". The power comes from giving them different boxes and then a composite operator that combines them: 

padding: 1.5px;
mask:
  linear-gradient(#000 0 0) content-box,   /* opaque only INSIDE the padding */
  linear-gradient(#000 0 0);               /* opaque over the WHOLE element  */
mask-composite: exclude;                   /* keep where exactly one is opaque */
  • Layer A is clipped to content-box — the region inside the 1.5 px padding. Opaque there, transparent in the padding band.
  • Layer B has the default border-box — opaque over everything, padding included.
  • mask-composite: exclude is set subtraction (a logical XOR): keep pixels opaque in exactly one layer. Inside the padding both are opaque → cancels → erased. In the padding band only B is opaque → kept. Result: just the padding band survives = a frame.

padding is now the ring thickness, and because masks respect border-radius, the frame follows the rounded corners automatically. Two knobs, one frame:

Why two spellings in the real code. You'll see both mask-composite: exclude and -webkit-mask-composite: xor in liquid-glass.tsx. Same operation, two vocabularies: the modern standard uses the keyword exclude; older WebKit/Blink shipped a different property whose equivalent keyword is xor. Both are kept so the ring renders across browser versions.

Boundaries

  • Solid-colour border, square or simple-radius corners? Just use border or outline. The mask trick is overkill unless you need a gradient ring on a rounded shape.
  • The ring's colour and where the light sits — that's lesson 0003. This lesson only carves the shape; that one paints it.
  • Distorting the background through the glass — unrelated; that's the SVG displacement layer from lesson 0002.
