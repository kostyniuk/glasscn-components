Glossary

Every lesson uses these terms with exactly these meanings. Lessons link here; look terms up, don't re-derive them.

Filter primitive (fe* element)
One pixel machine inside a <filter>: feGaussianBlur, feOffset, feColorMatrix, feMerge, feImage, feDisplacementMap… [Lesson 0001]
Assembly line
Our name for a <filter>: machines chained top-to-bottom; the filtered element displays only the LAST machine's output. [Lesson 0001]
in / result
The wiring. A machine labels its output result="name"; later machines consume it with in="name" (or in2 for a machine's second input). A machine with no in takes the previous machine's output. [Lesson 0001]
SourceGraphic / SourceAlpha
Free inputs available to any machine: the filtered pixels in full color / just their silhouette (black, same transparency). [Lesson 0001]
filter: url(#id) vs backdrop-filter: url(#id)
Same assembly line; different raw material. filter feeds in the element's own pixels (works in all browsers). backdrop-filter feeds in the live page BEHIND the element (SVG filters here render in Chromium only). [Lessons 0001–0002]
feImage
A machine that loads an external image (for us, a data-URL PNG) and places it on the line as a result. [Lesson 0002]
Displacement map
An image used as instructions, never shown: per pixel, Red = horizontal reach, Green = vertical reach, 128 = "don't move". A grid of arrows wearing a PNG costume. [Lesson 0002]
feDisplacementMap / scale
The machine that bends: samples in at (x + scale·(R/255 − 0.5), y + scale·(G/255 − 0.5)). Max reach = scale/2 px. The component's refraction prop sets scale. [Lesson 0002]
Texel
One pixel OF the map (as opposed to a screen pixel). The map can be lower-resolution than the element; it gets stretched to fit. [Lesson 0002]
Bezel
The band along the shape's edge where all refraction lives. In the component: a fraction (default 0.34) of half the smaller dimension. [Lesson 0002]
Convex squircle
The falloff curve (1 − (1−x)⁴)^¼ shaping how arrow strength fades from rim (strong) to interior (zero) — Apple's characteristic profile. [Lesson 0002, deeper in the SDF lesson]
SDF (signed distance function)
Math that answers, for any point: how far to the shape's edge, and in which direction? Used to paint the arrows for a rounded rectangle. Negative inside / positive outside by our convention's sign flip (inside = −d). [future lesson]
Capability gate / frosted fallback
The UA check (supportsSvgBackdropFilter()) deciding between real refraction (Chromium) and blur(blur+2) saturate() frost (Safari/Firefox/SSR). Needed because CSS.supports() false-positives. [future lesson]
Specular rim
The thin bright ring suggesting a light source — a separate <span> overlay, not part of the SVG filter. Built from two parts: the gradient paint and the mask ring (below). Lesson 0003
Layered linear-gradient (rim paint)
A linear-gradient colours by position on ONE axis and ignores the other. Stacking a to bottom (lights top/bottom) over a to right (darkens sides) aims the rim's light per edge; background layers paint front-to-back, first wins. Lesson 0003
Mask ring (mask-composite)
Carves the rim overlay into a thin frame: two opaque linear-gradient(#000 0 0) mask layers — one clipped to content-box, one to the whole box — combined with mask-composite: exclude (legacy -webkit-…: xor). The overlap inside the padding cancels, leaving only the padding band. padding = ring thickness; follows border-radius for free. Lesson 0004
