The LiquidGlass Pipeline

Compressed essence of components/ui/glasscn/liquid-glass.tsx. For the why, see Lesson 0001 → Lesson 0002, the glossary, and plans/001-liquid-glass-apple-parity.md.

Pipeline

mount → ResizeObserver measures (w, h) + computed borderTopLeftRadius
  → createDisplacementMap(w, h, r, bezel) — rounded-rect SDF → canvas → PNG data-URL (cached)
  → <feImage href=map> → <feDisplacementMap scale={refraction} R→X G→Y> → <feGaussianBlur 0.15>
  → backdrop-filter: url(#id) blur(blur) saturate(saturation) (Chromium only)
  → fallback: blur(blur + 2) saturate(saturation) (Safari / Firefox / SSR / first client render)
  + specular rim: absolutely-positioned span, 135° white gradient, 1.5px ring via mask-composite exclude

Rounded-rect SDF (per texel)

qx = |px| − (w/2 − r);  qy = |py| − (h/2 − r)
corner (qx>0 ∧ qy>0):  d = hypot(qx,qy) − r;  dir = sign(p)·q/len   // radial
edge   (else):          d = max(qx,qy) − r;    dir = axis of the max // straight
inside = −d   (> 0 inside the contour)
t = min(1, inside / bezelPx);  bezelPx = bezel · min(w,h)/2
magnitude = (1 − convexSquircle(t)) · min(1, inside / 1.25)   // edge taper
R = 128 + dirX·magnitude·127;  G = 128 + dirY·magnitude·127
convexSquircle(x) = (1 − (1−x)⁴)^¼   // steep at rim → flat interior

Decision log (rejected approaches)

ApproachWhy rejected
Chromatic aberration (3-pass R/G/B split)Apple's real material shows no RGB fringing — verified against macOS 26 Control Center screenshots. Common in web recreations; artistic license, not parity.
Snell's-law-exact ray-traced profileVisually indistinguishable from convexSquircle at UI element sizes; much harder to maintain.
WebGL/shader reimplementationShaders can't sample the live page behind an element; only backdrop-filter can.
CSS.supports() capability checkFalse-positives in Safari and Firefox (they parse but don't render).

Known limitations

  • Per-corner and percentage radii unsupported (reads borderTopLeftRadius in px only).
  • Radius-only changes without a size change don't re-measure (ResizeObserver doesn't fire).
  • Map cache is unbounded — continuous size animation thrashes it; animate transform: scale instead.
  • No supersampling in SVG displacement → rim pixelates at very high scale.
